# 3D-Kinematics
My 3D forward and inverse kinematics calculation programs.
The forward kinematics programs builds a virtual 3D arm using user input. The user can add as many joints or arms as they please. However, they must specify the angle (with the axis) of the joint or the length of the arm. After receiving all the input values, the program uses matrix calculations and matrix multiplication to produce a 3D coordinate that represents the location of the end position of the virtual arm.
The inverse kinematics program takes a 3-armed and 3-jointed virtual body and has the user input the end coordinate of the arm. Then, the program approximates the angles of the joints by taking a sample angle (starting from 0). Then, calculating its end-position before adjusting the angles accordingly. The program repeats this process until the end-position produced by the sample angles are less than the distance of error. However, if the loop iterates over a certain number (in this case 10,000), then the program states that it cannot reach the angle.
